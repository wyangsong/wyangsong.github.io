JVM专题

Created2026-01-01|Updated2026-01-04
|Post Views:

JVM内存区域
├── 线程共享区
│ ├── 堆(Heap) - 所有对象实例、数组
│ │ ├── 年轻代(Young Generation, 1/3堆大小)
│ │ │ ├── Eden区(8/10年轻代)
│ │ │ ├── Survivor From区(1/10年轻代)
│ │ │ └── Survivor To区(1/10年轻代)
│ │ └── 老年代(Old Generation, 2/3堆大小)
│ └── 元空间(Metaspace) - 类信息、常量池、方法数据
│
└── 线程私有区
├── 程序计数器(PC Register) - 当前线程执行的字节码行号
├── 虚拟机栈(VM Stack) - Java方法执行的内存模型
│ └── 栈帧(Stack Frame)
│ ├── 局部变量表
│ ├── 操作数栈
│ ├── 动态链接
│ └── 方法出口
└── 本地方法栈(Native Method Stack) - Native方法调用

1.堆(Heap)
频繁Full GC：老年代空间不足或元空间溢出

JVM参数调优示例（银行系统常见配置）

-Xms4g -Xmx4g # 堆初始和最大大小（生产环境通常相等避免扩容开销）
-Xmn2g # 年轻代大小（年轻代:老年代 ≈ 1:2）
-XX:SurvivorRatio=8 # Eden:Survivor=8:1:1
-XX:MaxTenuringThreshold=15 # 对象晋升老年代年龄阈值
-XX:+UseG1GC # 使用G1收集器（推荐）
2. 虚拟机栈(VM Stack) - 方法执行的生命周期
StackOverflowError：递归过深或局部变量过多
调整栈大小：-Xss256k（默认1M，银行系统可适当减小增加线程数）
3. 元空间(Metaspace) - 类加载的仓库
热部署频繁：每次部署都加载新类，旧类不卸载
-XX:MetaspaceSize=256m # 初始大小
-XX:MaxMetaspaceSize=512m # 最大大小（不设限可能导致内存泄漏）
-XX:+UseCompressedClassPointers # 压缩类指针（64位系统默认开启）
